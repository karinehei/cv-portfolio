### 2. CV version (4–5 lines)

### 3. Job application paragraph

For my **CV Portfolio QA Automation Project**, I built a personal portfolio website and approached it the same way I would any product in an agile team: I wrote a test plan and manual test cases, logged defects in a structured format, and automated regression checks with Robot Framework (UI) and Postman/Newman (API). I also ran baseline load tests with JMeter and accessibility evaluations with Lighthouse, Axe, and WAVE. All documentation follows Jira/Xray-style conventions, and a GitHub Actions workflow runs lint, build, API, and UI tests on each push. The project reflects how I work with developers and product owners — clarifying requirements, testing early, and documenting results clearly for stakeholders.

### 4. LinkedIn / GitHub project description

**CV Portfolio QA Automation Project**

A personal CV portfolio website used as a tested product for software tester / QA engineer job applications.

**What it demonstrates:**
- Manual testing — test plan, 15 test cases, structured bug reports, test summary report
- UI automation — Robot Framework smoke suite (Browser library)
- API testing — Postman collection executed with Newman in CI
- Load testing — JMeter baseline plan for home page and contact API
- Accessibility — Lighthouse, Axe, and WAVE workflows (WCAG 2.1 AA target)
- CI/CD — GitHub Actions: lint, typecheck, build, API + UI tests on every push

**Stack:** Next.js, TypeScript · **QA tools:** Robot Framework, Postman, JMeter, Lighthouse, Axe, WAVE

The repository includes full QA artefacts — not just the application — showing a practical testing mindset suitable for collaborative agile work with developers and product owners.
